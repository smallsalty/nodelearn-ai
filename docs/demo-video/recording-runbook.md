# 逐镜头录制手册

## 录制前门禁

1. 使用 `demo_student / demo_password` 登录，课程选择“Hello 算法”。
2. 检查首页、`/api/v1/system/health`、课程图谱、问答历史、资源详情、Judge0 和笔记 API。
3. 健康结果允许旧 `iflytekTts=error`；知识视频使用豆包 TTS，该旧健康项不参与本次链路。
4. 确认数字人聊天和在线虚拟人均为 `ok`，FFmpeg 无残留会话。
5. 确认哈希表视频是审核通过的历史资源；不得点击正式付费视频生成。
6. 后端若重启，先重新生成学习路径与编程题，因为两者当前仍使用进程内存存储。

## 固定输入

### 问答助手

> 请结合当前课程材料，用不超过150字和一个餐盘例子解释栈的后进先出，并指出两个容易混淆的操作。

### 数字人解答

> 请结合当前课程材料，用不超过150字和一个餐盘例子解释栈的后进先出。

### 视频讲解目标

> 用动态演示讲清哈希函数、桶定位、查找复杂度和哈希冲突，并给出一个具体查找示例。

### 学习笔记

```markdown
## 栈的核心结论
- 栈遵循后进先出，入栈和出栈都发生在栈顶。
- 数组栈扩容时需要区分单次复杂度和摊销复杂度。
```

## 镜头清单

### S01 开场，00:00–00:20

- 使用标题卡“NodeLearn AI——多智能体个性化学习平台”。
- 依次闪现知识图谱、思维导图、教学视频、数字人，各 2–3 秒。
- 结尾切到登录页。

### S02 登录与首页，00:20–00:45

- 登录页停留 3 秒，点击“登录”。
- 首页停留在课程、画像摘要、路径和推荐资源区域。
- 不朗读首页具体百分比，避免演示数据变化影响口播。

### S03 知识图谱，00:45–01:20

- 点击“课程管理 / 知识节点”。
- 展开包含哈希表的章节，选择“哈希表”。
- 展示节点依赖、难度、掌握状态以及“查看正文 / 课后测试 / 生成思维导图 / 视频讲解”。
- 关键验收：页面不得出现内部 `nodeId`。

### S04 画像与学习路径，01:20–02:00

- 学生画像停留 8 秒，展示基础、偏好和薄弱点。
- 学习路径展示 6 项任务、建议时间、工具卡和中文提示词。
- 若路径为空，录制前重新生成；不要在成片中展示 3 分钟等待。

### S05 RAG 问答，02:00–02:50

- 进入问答助手，发送固定问题。
- 保留点击和“正在结合课程材料思考”状态，随后跳剪到真实回答。
- 画面显示“真实生成过程已加速”2 秒。
- 展示问答历史已经刷新，不展示认证请求头。

### S06 思维导图，02:50–03:25

- 将当前节点切回“哈希表”，进入“思维导图”。
- 可展示一次提交和任务状态，再切到审核通过的导图。
- 点击“全部展开”、一个分支和“聚焦当前节点”。
- 关键验收：导图可交互、标题中文、无原始 JSON 泄漏。

### S07 教学视频，03:25–04:15

- 进入“视频讲解”，从历史资源选择器打开最新合格视频。
- 展示 `script → storyboard → quality_audit → tts → render → audit → persist` 标签。
- 播放 12–15 秒，优先使用桶定位、查找对比和冲突演示。
- 不点击“生成讲解视频”，不展示两条已撤回的约 62.7 秒失败资源。

### S08 数字人，04:15–05:15

- 节点选择“5.1 栈”，进入“数字人解答”。
- 发送固定问题，保留 2 秒启动状态后跳剪到直播画面。
- 保留 12–15 秒带声音播放，展示回答下的课程材料标签。
- 最后点击“结束会话”，确认状态不再是直播中。

### S09 Judge0 编程题，05:15–06:00

- 进入“练习测评 / 编程题”，选择“循环左移数组”。
- 展示公开样例和 Python 代码，点击运行并显示 AC、时间和内存。
- 可插入 2 秒 WA 画面，禁止展示隐藏用例。

### S10 学习笔记，06:00–06:25

- 打开“学习笔记”，新建“栈与数组栈复习”。
- 输入固定 Markdown 并保存，展示“已与服务端同步”。
- 只保留与演示相关的标签和关联，不删除用户已有笔记。

### S11 总结，06:25–06:45

- 回到首页，点击推荐区中的合格视频。
- 切到总结卡：多智能体、课程材料 RAG、多模态视频、实时数字人、安全审核。
- Logo 和项目名停留至少 2 秒。
